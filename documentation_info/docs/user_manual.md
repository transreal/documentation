# Documentation パッケージ ユーザーマニュアル

## 概要

Documentation パッケージは、Mathematica ノートブック上で **LLM を活用した文書作成・翻訳支援** を行うツールです。

### 設計思想

このパッケージの核心は「**アイデアメモから文章品質のパラグラフへ**」というワークフローです。

文章を書くとき、最初から完成した文を書こうとすると手が止まりがちです。このパッケージは「まず箇条書きや断片的なメモ（アイデア）を書き、LLM に整形を委ねる」というアプローチを採用しています。書き手はアイデアの内容に集中し、文章の流暢さや翻訳はツールが担当する、という分業が基本思想です。

もうひとつの設計原則は **可逆性** です。LLM が生成したパラグラフは保存されますが、元のアイデアメモも常に保持されます。ボタン一つでアイデアとパラグラフを行き来でき、いつでも書き直し・再展開が可能です。

さらに、**多言語対応** を第一級機能として位置づけています。日本語で書いた文章を英語に翻訳し、元文を編集したら翻訳を更新する、というサイクルをパレットのボタン操作だけで完結できます。

### 主な機能

- **アイデア展開**: 断片的なメモを文章品質のパラグラフに変換
- **表示切替**: アイデアとパラグラフをワンクリックで行き来
- **翻訳**: パラグラフを指定言語に翻訳
- **同期** *(API 経由)*: 元文を編集したとき `DocSync` API で翻訳を最新の状態に更新
- **コード生成（計算）**: セルに書いたプロンプトから実行可能な Wolfram Language コードを自動生成
- **セル分割・合併**: 1つのセルを前後に分割、または複数セルを1つに統合
- **×展開**: セルに保存された展開データを削除してアイデアメモの状態に戻す
- **×翻訳**: セルに保存された翻訳データを削除して翻訳前の状態に戻す
- **×計算**: セルに生成されたコードを削除してプロンプト状態に戻す
- **一括処理**: ノートブック全体を対象にした一括表示切替・一括翻訳
- **Note セル**: エクスポート対象外のメモ・注釈セルを挿入
- **Directive セル**: 展開・翻訳・同期・計算時に LLM が従うべき指示を記載するセルを挿入
- **Dictionary セル**: 翻訳時の用語対応を指定する辞書セルを挿入
- **Bibliography セル**: 参考文献リストを管理するセルを挿入
- **図メタデータ**: 図のラベル・キャプションを設定し、本文から `<<fig:label>>` で参照
- **参照挿入**: 図・文献参照のマーカーをダイアログから本文に挿入
- **依存資料**: セルの内容生成に使用した PDF 資料とページ番号を記録
- **自動引用挿入**: 依存資料から文献リストを構築し、本文に引用マーカーを自動挿入
- **除外切替**: 選択セルをエクスポート対象外に設定/解除
- **エクスポート** *(試験的)*: ノートブックを Markdown、LaTeX、または MS-Word 形式で出力（+Math オプションで数式を LLM により自動フォーマット）

## LMStudio ローカルモデルの使用（オプション）

Anthropic の Claude API の代わりに、**LMStudio** で起動したローカル LLM を使用することができます。インターネット接続が不要なため、機密性の高い文書の執筆や、API コストを抑えたい場合に有効です。

> **参照**: LMStudio 連携の詳細な仕組みは NBAccess パッケージおよび claudecode パッケージのドキュメントも合わせてご参照ください。

### 設定手順

1. [LMStudio](https://lmstudio.ai/) をインストールし、使用したいモデル（例: `qwen/qwen3.6-27b`）をダウンロードして起動します。LMStudio の「Local Server」機能を有効にしてください（デフォルトポート: `1234`）。

2. Mathematica の初期化コード（`init.m` または各ノートブックの冒頭）に以下の設定を追記します。

```mathematica
(* LMStudio で利用可能な MCP 統合を指定 *)
$ClaudeLMStudioIntegrations = {"mcp/exa"};

(* ローカルモデルの設定: {プロバイダ, モデル名, ベースURL} *)
$ClaudePrivateModel = {"lmstudio", "qwen/qwen3.6-27b", "http://127.0.0.1:1234"};

(* $ClaudeModel をローカルモデルに切り替える *)
$ClaudeModel = $ClaudePrivateModel;
```

| 設定値 | 説明 |
|---|---|
| `$ClaudeLMStudioIntegrations` | LMStudio モード時に有効化する MCP ツールのリスト。`{"mcp/exa"}` は Exa 検索 MCP を有効化します |
| `$ClaudePrivateModel` | `{プロバイダ種別, モデル名, ベースURL}` の 3 要素リスト |
| `$ClaudeModel` | `$ClaudePrivateModel` を代入するとローカルモデルに切り替わります |

3. パッケージをロードします。

```mathematica
Needs["Documentation`"]
```

### 元の Claude API に戻す方法

```mathematica
(* $ClaudeModel を文字列に設定すると Anthropic API に戻る *)
$ClaudeModel = "claude-sonnet-4-20250514"
(* または空文字列でデフォルトモデルを使用 *)
$ClaudeModel = ""
```

### 注意事項

- LMStudio のサーバーが起動していない場合、LLM 呼び出しはエラーになります。パレットを使用する前に LMStudio のローカルサーバーが起動していることを確認してください。
- ローカルモデルの性能は Claude API と異なる場合があります。特に翻訳・展開の品質はモデルの能力に依存します。
- パレットの **課金 API** 設定（On/Off）は Anthropic API 向けの設定です。`$ClaudeModel` をリストに設定した場合、課金 API の設定に関わらず LMStudio が使用されます。
- `$ClaudePrivateModel` の第3要素（ベースURL）は LMStudio のサーバー設定に合わせて変更してください。

## クイックスタート

パッケージをロードすると、パレットが自動的に表示されます。メニューからも呼び出せます。

```mathematica
Needs["Documentation`"]
```

パレットが表示されたら、ノートブックのセルにアイデアメモを書いて **▶ 展開** ボタンを押すだけで文書作成が始まります。

---

## パレットの使い方

パレットは **Documentation** というタイトルのフローティングウィンドウで表示されます。ノートブックとは独立したウィンドウとして常に手前に表示されるため、編集中のノートブックを操作しながら使えます。

### 基本的な操作の流れ

1. ノートブックのセルにアイデア（メモ・箇条書き・断片的な文）を書く
2. そのセルをクリックして選択する
3. パレットのボタンを押して操作を実行する

セルが選択されていない場合は、直前に操作したセルが自動的に再利用されます。

---

### 執筆ツール ボタン

パレットの執筆ツールエリアでは、関連するボタンが2列の行として並んでいます。主要な操作ボタンと、対応するデータ削除ボタンが横に並んでいます。

#### ▶ 展開（Expand）／ ×展開（×Exp）

**▶ 展開** は、選択したセルに書かれたアイデアメモを、LLM が文章品質のパラグラフに書き直します。

- セルに書いたメモが短い断片であっても、自然な文章に整形されます
- ノートブック内の **Directive セル**（後述）や **Dictionary セル**（後述）の内容が自動的に LLM への指示・用語辞書として参照されます
- 展開後もアイデアメモは内部に保存されるため、**再展開や表示切替がいつでも可能**です
- **既にパラグラフ展開済みのセルに対して再度押すと、現在のパラグラフ文章・Directive セルの指示・文脈を踏まえてパラグラフをインプレースで更新します。** Directive や Dictionary の内容を追加・変更した後に既存パラグラフを一括改善したい場合に使います
- 複数のセルを選択している場合、メタセル（Note・Directive・Dictionary など）をスキップして順に処理します。展開済みのセルも更新対象になります

**×展開** は、選択したセルに保存されている展開データ（アイデアメモ・パラグラフの対応）を削除します。

- 実行前に確認ダイアログが表示されます。**この操作は元に戻せません**
- パラグラフ表示中のセルは、削除前にアイデアメモの内容をセルに書き戻してから削除します
- 削除後はセルが通常のテキストセル（展開前の状態）に戻り、枠線スタイルもリセットされます

**使いどころ（展開）**: 箇条書きのメモから論文・報告書・ドキュメントの本文を生成したいとき、または既存のパラグラフを改めて改善・書き直したいとき。  
**使いどころ（×展開）**: 誤って展開したセルをリセットしたいとき、またはセルを展開前の素のテキストセルとして扱い直したいとき。

---

#### » 翻訳（Translate）／ ×翻訳（×Tr）

**» 翻訳** は、選択したセルのパラグラフを、`$DocTranslationLanguage` で指定した言語に翻訳します。

- アイデア（プロンプト）モードのセルはスキップされます（パラグラフ展開後に翻訳してください）
- 翻訳後は自動的に翻訳表示に切り替わります
- **Dictionary セル**（後述）が存在する場合、指定した用語対応に従って翻訳されます
- **Directive セル**（後述）の指示も翻訳時に適用されます
- **既に翻訳済みのセルに対して再度押すと、現在のパラグラフから翻訳を再生成します。** Dictionary や Directive の内容を更新した後に翻訳をやり直したい場合に便利です
- 複数セルを選択した場合、アイデアモードのセルを飛ばして順に翻訳します。翻訳済みのセルも再翻訳の対象になります
- 翻訳先の言語はデフォルトで英語（Mathematica の言語設定が英語の場合は日本語）ですが、変更可能です

```mathematica
(* 翻訳先をフランス語に変更する例 *)
$DocTranslationLanguage = "French"
```

**×翻訳** は、選択したセルに保存されている翻訳データを削除します。

- 実行前に確認ダイアログが表示されます。**この操作は元に戻せません**
- 翻訳表示中のセルは、削除前に元パラグラフの内容をセルに書き戻してから削除します
- 削除後は翻訳タグがすべてクリアされ、セルがパラグラフ（または展開前）状態に戻ります

**使いどころ（翻訳）**: 日本語で書いた文章を英語論文・英語ドキュメントに変換したいとき、または既存の翻訳を更新・やり直したいとき。  
**使いどころ（×翻訳）**: 翻訳をやり直したいとき、あるいは翻訳データを含まないクリーンな状態にしたいとき。

---

#### 計算（Compute）／ ×計算（×Cmp）

**計算** は、選択したセルに書かれたプロンプトから、LLM が実行可能な Wolfram Language / Mathematica コードを生成します。

- セルのテキストをプロンプトとして解釈し、対応するコードを生成してセルに書き込みます
- 生成後はセルが**オレンジ色の枠線**で表示されます（計算モード）
- ノートブック内の **Directive セル** の内容が LLM への指示として参照されます
- ノートブックのコンテキスト（文脈情報）が自動収集され、略語や固有名詞の解釈に使われます
- **コード表示中（計算モード）のセルに再度押すことはできません。** **↔ 切替** でプロンプト表示に戻してからプロンプトを編集し、再実行してください
- 複数セルを選択した場合、約 2 秒間隔の非同期チェーンで順に処理します

**×計算** は、選択したセルに生成されたコードを削除し、プロンプト状態に戻します。

- 実行前に確認ダイアログが表示されます。**この操作は元に戻せません**

**使いどころ（計算）**: セルに書いた内容の計算・グラフ描画・データ処理に対応するコードを素早く生成したいとき。  
**使いどころ（×計算）**: 生成されたコードをリセットしてプロンプトから書き直したいとき。

---

#### 分割（Split）

カーソル位置で選択したセルを前半・後半の2つのセルに分割します。

- パラグラフ・翻訳表示中の場合、表示テキストと保存データ（アイデア・翻訳）をそれぞれ対応位置で分割します
- プロンプト（アイデア）が存在する場合、LLM で前半・後半用のプロンプトを再生成します
- 通常のテキストセルの場合は単純にテキストを分割します

**使いどころ**: 一つのセルが長くなりすぎたとき、段落を分けたいとき。

---

#### 合併（Merge）

選択した複数のセルを1つのセルに統合します。

- テキスト・プロンプト・翻訳をそれぞれ結合し、最初のセルに統合します
- モード・スタイルは最初のセルを維持します

**使いどころ**: 短いセルが並んでいる部分をまとめたいとき。

---

#### ↔ 切替（Toggle）

選択したセルの表示を切り替えます。

- **アイデアメモ ↔ パラグラフ**: 通常の展開済みセルの場合
- **パラグラフ ↔ 翻訳**: 翻訳済みセルの場合
- **プロンプト ↔ コード**: 計算モードのセルの場合

- 現在表示中の内容（編集していた場合もその内容）を保存してから切り替えます
- アイデアメモを直接編集した後に展開し直したい場合は、アイデア表示に戻してから内容を修正し、改めて **▶ 展開** を押します

**使いどころ**: 元のメモを確認したいとき、生成されたパラグラフとメモを見比べたいとき、または計算のプロンプトを編集したいとき。

---

#### ■ メモ（Note）

現在のカーソル位置に **Note スタイル**のセルを挿入します。

- Note セルはノートブック上に通常のテキストとして表示されますが、**Markdown・LaTeX・Word エクスポート時には本文に含まれません**
- 執筆中のメモ・TODO・コメントなど、最終出力には残したくない注釈を書くのに適しています
- ノートブックに `"Note"` スタイルが既に定義されている場合はそれを使用し、なければパッケージ組み込みのカスタム Note セルを挿入します

**使いどころ**: 「ここは後で書き直す」「参考文献を確認する」など、執筆中のメモを本文と混在させて置いておきたいとき。

---

#### 指示（Directive）

現在のカーソル位置に **Directive スタイル**のセルを挿入します。

- Directive セルには、展開・翻訳・同期・計算の実行時に **LLM が順守すべき指示**を自由に記載できます
- ノートブック内に複数の Directive セルを配置できます。すべての Directive セルの内容が LLM に渡されます
- エクスポート時には本文に含まれません

**形式の例**:
```
- 専門用語は必ず太字にする
- 文末は「です・ます調」で統一する
- 箇条書きは3項目以内にまとめる
```

**使いどころ**: 文体・表記ルール・構成方針など、ノートブック全体に適用したい執筆指針を一元管理したいとき。

---

#### 辞書（Dict）

現在のカーソル位置に **Dictionary スタイル**のセルを挿入します。

- 翻訳時に **用語の対応**（例: 日本語 → 英語）を指定するための辞書です
- 翻訳・同期の実行時に自動的に参照され、指定した用語が一貫して翻訳されます
- エクスポート時には本文に含まれません

**形式**（GridBox 形式のテーブル）:

```
{{<<Japanese>>, <<English>>, <<Context>>},
 {"展開", "expand", "パラグラフ展開機能"},
 {"翻訳", "translate", "言語変換機能"},
 {"同期", "sync", "翻訳の更新"}}
```

- 1行目はヘッダー（`<<>>` で囲む）
- 2行目以降が実際の用語対応。Context 列で同音異義語の使い分けを指定できます

**使いどころ**: 技術用語・固有名詞の翻訳を統一したいとき。

---

#### 文献（Bib）

現在のカーソル位置に **Bibliography スタイル**のセルを挿入します。

- 参考文献リストを管理するためのセルです
- 本文中で `<<cite:key>>` と記述すると、エクスポート時に Markdown では `[key]`、LaTeX では `\cite{key}` に自動変換されます
- **自動引用挿入**（後述）を実行すると、Bibliography セルが自動生成・更新されます
- エクスポート時には参考文献セクションとして出力されます（本文とは別に処理）

**形式**:

```
{{<<Key>>, <<Author>>, <<Year>>, <<Title>>},
 {"morita1996", "Morita K", "1996", "Reversible simulation of one-dimensional irreversible cellular automata"},
 {"wolfram2002", "Wolfram S", "2002", "A New Kind of Science"}}
```

---

#### ■ 図メタ（Fig Meta）

選択した画像セルの **図メタデータ**（ラベルとキャプション）を編集するダイアログを表示します。

- **ラベル**（参照用キー）: 本文から `<<fig:label>>` と記述して参照します。LaTeX エクスポート時は `\ref{fig:label}` に変換されます
- **キャプション**: 図の下に表示される説明文です
- 設定後はエクスポート時に `figure` 環境（LaTeX）または適切な Markdown 形式で出力されます

**本文中での参照例**:
```
図 <<fig:architecture>> にシステムの全体構成を示します。
```

**使いどころ**: 図番号を自動管理しながら本文から図を参照したいとき。

---

#### ▶ 参照挿入（Ref Insert）

図参照（`<<fig:label>>`）または文献引用（`<<cite:key>>`）のマーカーをダイアログから選んで本文に挿入します。

- ノートブック内の図メタデータと Bibliography セルの内容を一覧表示し、選択して挿入できます
- 手動でマーカーを入力する手間を省きます

---

#### ■ 依存資料（Ref Src）

選択したセルの **依存資料**（参照元 PDF）を設定するダイアログを表示します。

- そのセルの内容生成に使用した PDF ファイルと参照ページ番号を記録します
- LaTeX + Math エクスポート時に、該当ページのみを LLM に送付して **トークン消費を削減**します
- **自動引用挿入**（DocAutoInsertCitations）の引用元情報としても使われます

---

#### ▶ 自動引用（Auto Cite）

ノートブック内のすべてのセルに **自動引用マーカーを挿入**します。

処理は以下のフェーズで非同期実行されます：

1. **文献情報収集**: 各セルの依存資料（Ref Src）を収集し、PDF ファイル名から著者・年・タイトルを解析します
2. **書誌情報の精緻化**: PDF テキストを LLM に送り、著者名・タイトル・掲載誌などを正確化します
3. **引用挿入**: 各セルのパラグラフに最も適切な位置に `<<cite:key>>` を1つ挿入します
4. **未引用文献の配置**: いずれのセルにも引用されなかった文献を、関連するパラグラフに追加配置します（無関係と判断した場合は UNRELATED としてマーク）
5. **Bibliography セル生成/更新**: 収集した文献情報で Bibliography セルを自動生成または更新します

> **注意**: 依存資料（Ref Src）が設定されたセルがない場合、自動引用は実行されません。

**使いどころ**: 参考文献を大量に扱う論文執筆で、引用の挿入と文献リストの管理を自動化したいとき。

---

#### × 除外切替（Excl Toggle）

選択したセルをエクスポート対象外に設定、または解除します。

- Note・Directive・Dictionary・Bibliography 以外のセルに対して、個別にエクスポート除外を設定できます
- 除外されたセルはノートブック上では通常通り表示されますが、Markdown・LaTeX・Word エクスポート時に本文から除かれます
- 再度ボタンを押すと除外設定が解除されます

**使いどころ**: 特定のセルだけ出力に含めたくない場合（例: 草稿段階のセクション、計算結果の中間セル）。

---

### 一括表示切替 ボタン

ノートブック全体のセルの表示を一括で切り替えます。

#### … 全プロンプト（All Prompts）

ノートブック内のすべての Documentation セルをアイデア（プロンプト）表示にします。

**使いどころ**: 文章の構成や内容のアウトラインを一覧で見直したいとき。

#### ¶ 全パラグラフ（All Paragraphs）

ノートブック内のすべてのセルをパラグラフ表示にします。

**使いどころ**: 生成された文章全体を通して読みたいとき、印刷・エクスポート前の最終確認。

#### Â 全翻訳（All Translations）

翻訳済みのセルをすべて翻訳表示に切り替えます。未翻訳のセルはそのままです。

**使いどころ**: 英語版の文書全体を通して読みたいとき。

---

### エクスポート ボタン *(試験的)*

> **注意**: エクスポート機能は現在試験的な実装です。複雑なノートブックでは変換結果が不完全になる場合があります。

#### → Markdown

現在のノートブックを Markdown 形式でエクスポートします。`DocExportMarkdown` を呼び出します。

- 出力先: `NotebookDirectory[] / <ノートブック名>_md/`
- **Note・Directive・Dictionary・Bibliography スタイルのセルおよび除外設定されたセルは出力から除外**されます
- ラスター画像は PNG、ベクター画像・計算結果は PDF で保存されます
- Input セルはコードブロック（` ```mathematica ``` `）として出力されます
- 数式は TeX 形式に変換されます
- `<<fig:label>>` は `{#fig-label width=100%}` 形式の図参照に変換されます
- `<<cite:key>>` は `[key]` に変換されます

**使いどころ**: GitHub や静的サイトジェネレータ向けにノートブックの内容を出力したいとき。

#### → LaTeX / +Math

現在のノートブックを LaTeX 形式でエクスポートします。

- **LaTeX**: 標準の LaTeX エクスポートです。`DocExportLaTeX` を呼び出します
- **+Math**: 数式自動フォーマット付きエクスポートです。`DocExportLaTeX[nb, "MathFormat" -> True]` を呼び出します。各テキストセル内の数式を LLM が解析し、適切な LaTeX 数式表現（インライン `$...$` またはディスプレイ `\[...\]`）に自動変換します。Box 式や FormBox を含む Mathematica の数式表現も LLM によって適切な LaTeX 形式に変換されます

- 出力先: `NotebookDirectory[] / <ノートブック名>_LaTeX/`
- **Note・Directive・Dictionary・Bibliography スタイルのセルおよび除外設定されたセルは出力から除外**されます
- 画像は本文幅いっぱい（`\includegraphics[width=\textwidth]`）で出力されます
- `<<fig:label>>` は `\ref{fig:label}` に、`<<cite:key>>` は `\cite{key}` に変換されます
- 参考文献は `thebibliography` 環境として末尾に自動出力されます

**使いどころ**: 論文・技術文書の LaTeX ソースとして仕上げたいとき。**+Math** は Mathematica の数式セルではなく、テキスト中に書かれた数式（例: 「f(x) = x^2 + 1」のような表現）を適切な LaTeX 形式にしたいときに使います。

#### → Word / +Math

現在のノートブックを Microsoft Word (.docx) 形式でエクスポートします。`DocExportWord` を呼び出します。

- 内部で Markdown エクスポートを実行し、**Pandoc** を使って `.docx` に変換します
- 出力先: `NotebookDirectory[] / <ノートブック名>_md/<ノートブック名>.docx`
- **+Math**: 数式自動フォーマット付きエクスポートです（LaTeX の +Math と同様の処理が Markdown 段階で適用されます）
- Pandoc がインストールされている必要があります（`setup.md` 参照）

```mathematica
(* テンプレート .docx を指定してエクスポートする例 *)
DocExportWord[EvaluationNotebook[], "ReferenceDoc" -> "/path/to/template.docx"]
```

**使いどころ**: Word で仕上げを行いたいとき、または Word 形式での提出が必要なとき。

---

### 設定

パレット下部の設定ボタンは ClaudeCode パレットと共有されます。片方で変更すると両方に反映されます。

#### モデル

使用する LLM モデルを切り替えます。クリックするたびに **Default → Opus → Sonnet → Default** と循環します。

- **Default**: タスクに応じて自動選択されます（通常はこれで十分です）
- **Opus**: 最高品質。文章の質を最大限に高めたいとき
- **Sonnet**: 高速・省コスト。大量のセルを一括処理したいとき

> **注意**: `$ClaudeModel` をリスト（LMStudio 形式）に設定している場合、パレットのモデル表示は参考値となり、実際には `$ClaudeModel` で指定したローカルモデルが使用されます。

#### エフォート

出力の品質レベルを切り替えます（**Low → Medium → High → Max → Low** と循環）。Sonnet モデルでは変更できません。

- 品質を上げるほど生成に時間がかかりますが、より丁寧な文章になります
- 通常は **Medium** で十分です

#### 課金 API

Claude API（有料の外部 API）の使用を **許可 / 禁止** します。

- 禁止（Off）のとき: ローカルの LLM またはデフォルトの無料 API を使用します
- 許可（On）のとき: Claude API を使用します。高品質な結果が得られますが、API 利用料が発生します
- `$ClaudeModel` をリスト（LMStudio 形式）に設定している場合、この設定に関わらず LMStudio が使用されます

---

### ステータス表示

パレット下部に現在のノートブックのセル状態サマリーが 2 秒ごとに更新されます。

| 表示 | 意味 |
|---|---|
| **展** / **E** | パラグラフ展開済みセルの数 |
| **ア** / **I** | アイデア（未展開）モードのセルの数 |
| **訳** / **T** | 翻訳済みセルの数 |

このステータスを見ることで、「あと何セル展開が必要か」「翻訳が完了しているか」を一目で確認できます。

## 典型的なワークフロー

### ワークフロー 1: アイデアメモから文書を作る

最もシンプルな使い方です。

1. ノートブックに新しいテキストセルを作り、アイデアや要点をメモとして書く

   ```
   - LLMを使った文書作成ツール
   - アイデア→パラグラフ展開が主機能
   - パレットUIで操作
   ```

2. そのセルを選択してパレットの **▶ 展開** をクリックする

3. LLM がメモを自然な文章に書き直す（数秒〜十数秒）

4. 文章を確認し、修正したい場合はセルを直接編集する

5. アイデアメモを修正して再展開したい場合は **↔ 切替** でアイデア表示に戻し、メモを編集してから再度 **▶ 展開**

---

### ワークフロー 2: 多言語ドキュメントを作る

日本語で書いた文書を英語に翻訳する典型的なフローです。

```mathematica
(* 翻訳先を英語に設定（デフォルトでも通常英語） *)
$DocTranslationLanguage = "English"
```

1. 日本語でアイデアメモを書いて **▶ 展開** でパラグラフを生成する

2. パラグラフ表示のまま **» 翻訳** をクリックして英語翻訳を生成する

3. **↔ 切替** で日本語と英語を切り替えながら内容を確認する

4. 日本語を修正したい場合は **↔ 切替** で日本語表示に戻して編集する

5. 修正後に **» 翻訳** を再実行すると英語翻訳が更新されます。元パラグラフの変更が軽微な場合は `DocSync[nb, cellIdx]` API で差分更新することもできます

6. 最終的に **¶ 全パラグラフ** または **Â 全翻訳** でノートブック全体を一方の言語で通し読みする

---

### ワークフロー 3: ノートブック全体を一括処理する

複数セルをまとめて展開・翻訳したいときのフローです。

1. `Shift + クリック` や `Ctrl + A` などでまとめて複数セルを選択する

2. **▶ 展開** または **» 翻訳** をクリックする

3. パレット下部のステータスで進捗を確認する（「展開中: 3/10」のように表示される）

4. 処理は約 2 秒間隔の非同期チェーンで実行されるため、処理中もノートブックの編集は可能

あるいは、すべてのセルに対して順に処理する場合は **¶ 全パラグラフ**・**Â 全翻訳** ボタンを使うと手動での複数選択が不要です。

---

### ワークフロー 4: 文書の見直しと再構成

1. **… 全プロンプト** でノートブック全体をアイデアメモ表示に切り替えて構成を見直す

2. セルの順序を並べ替えたり、新しいメモを追加する

3. **¶ 全パラグラフ** に切り替えて文章としての流れを確認する

4. 必要なセルだけ選択して **▶ 展開** で再展開する

---

### ワークフロー 5: 展開・翻訳データをリセットする

セルの状態を初期状態に戻したいときのフローです。

1. リセットしたいセルを選択する（複数選択可）

2. 翻訳データのみリセットする場合は **×翻訳** をクリックする

   - 確認ダイアログで **削除** を選ぶと翻訳データが削除され、パラグラフ状態に戻る
   - その後 **» 翻訳** で改めて翻訳を生成できる

3. 展開データごとリセットする場合は **×展開** をクリックする

   - 確認ダイアログで **削除** を選ぶと展開データがすべて削除され、素のテキストセルに戻る
   - その後 **▶ 展開** で改めて展開できる

> **注意**: 削除操作は元に戻せません。大切なパラグラフや翻訳は事前にコピーしておくことをお勧めします。

---

### ワークフロー 5b: コード生成（計算）を使う

セルのテキストをプロンプトとして Wolfram Language コードを生成するフローです。

1. テキストセルにプロンプトを書く

   ```
   1から100の素数をリストアップしてグラフィカルに表示する
   ```

2. そのセルを選択してパレットの **計算** をクリックする

3. LLM が対応する Wolfram Language コードを生成してセルに書き込む（オレンジ色の枠線で表示）

4. 生成されたコードを確認・編集して実行する

5. プロンプトを修正して再生成したい場合は **↔ 切替** でプロンプト表示に戻し、編集してから再度 **計算** をクリックする

6. コードを削除してプロンプト状態に戻したい場合は **×計算** をクリックする

---

### ワークフロー 6: 用語を統一して翻訳する

翻訳時に専門用語を一貫させたいときのフローです。

1. **辞書** ボタンで Dictionary セルを挿入し、用語対応を記入する

   ```
   {"展開", "expand", ""},
   {"翻訳同期", "translation sync", ""},
   {"アイデアメモ", "idea prompt", ""}
   ```

2. 通常通り **» 翻訳** を実行する（Dictionary セルは自動的に参照される）

3. 用語の翻訳が統一されていることを確認する

4. 用語対応を変更した場合は **» 翻訳** を再実行して翻訳を更新する

---

### ワークフロー 7: LLM への指示を設定する

文体・表記ルールなどをノートブック全体に適用するフローです。

1. **指示** ボタンで Directive セルを挿入し、LLM への指示を記入する

   ```
   - 文末は「です・ます調」で統一する
   - 専門用語は初出時にのみ説明を加える
   - 段落は3〜5文程度に収める
   ```

2. この Directive セルは展開・翻訳・同期・計算のすべての操作で自動的に参照される

3. ノートブックの別の場所に追加の Directive セルを置くことも可能（複数配置した場合、すべてが適用される）

---

### ワークフロー 7b: Directive・Dictionary を変更して既存の文章を更新する

執筆方針や用語辞書を後から変更し、既存のパラグラフや翻訳に反映させたいときのフローです。

1. Directive セルまたは Dictionary セルの内容を編集する

2. 更新したいセルを選択する（`Ctrl + A` で全選択も可能）

3. **▶ 展開** を押すと、現在のパラグラフ文章・更新済みの Directive・文脈を踏まえてパラグラフがインプレースで更新される

4. 翻訳も更新したい場合は続けて **» 翻訳** を押す。翻訳済みセルも再翻訳の対象になるため、更新された Dictionary や Directive が反映される

   - 元パラグラフの変更が最小限の場合は `DocSync[nb, cellIdx]` API を呼び出すことで差分更新もできます

---

### ワークフロー 8: 図を参照する

図に番号付きラベルを付けて本文から参照するフローです。

1. 画像セルを選択してパレットの **■ 図メタ** をクリックする

2. ダイアログで **ラベル**（例: `architecture`）と **キャプション**（例: `システムの全体構成`）を入力する

3. 本文のテキストセルに `<<fig:architecture>>` と記述して図を参照する

4. エクスポート時にラベルが適切な形式（LaTeX: `\ref{fig:architecture}`、Markdown: `{#fig-architecture width=100%}`）に変換される

---

### ワークフロー 9: 参考文献を管理する

文献リストを構築して本文中に引用するフローです。

#### 手動で文献を管理する場合

1. **文献** ボタンで Bibliography セルを挿入し、文献エントリを記入する

   ```
   {"morita1996", "Morita K", "1996", "Reversible simulation of ..."},
   {"wolfram2002", "Wolfram S", "2002", "A New Kind of Science"}
   ```

2. 本文中に `<<cite:morita1996>>` と記述して引用する

3. エクスポート時に引用が適切な形式に変換される（LaTeX では `\cite{morita1996}`）

#### 自動引用を使う場合

1. 各セルで **■ 依存資料** をクリックして、そのセルの内容生成に使用した PDF を指定する

2. **▶ 自動引用** をクリックする

3. 書誌情報の精緻化 → 引用挿入 → 未引用文献の配置 → Bibliography セル生成、の順に自動処理される

4. 生成された Bibliography セルや引用マーカーを必要に応じて手動で修正する

---

### ワークフロー 10: Markdown /LaTeX / Word にエクスポートする *(試験的)*

執筆が完了したノートブックを外部形式に書き出すフローです。

1. **■ メモ** ボタンで挿入した Note セルに執筆中のメモや TODO が残っていないか確認する（Note セルはエクスポートに含まれないため、残っていても問題ありません）

2. 最終的に出力に含めたくないセルがある場合は **× 除外切替** でエクスポート対象外に設定する

3. **¶ 全パラグラフ** または **Â 全翻訳** でノートブックを最終表示状態に整える

4. パレットの **→ Markdown**、**LaTeX**、または **Word** をクリックする

   - 数式を LLM で自動フォーマットしたい場合は **+Math** 付きのボタンを使用する

5. エクスポートが完了すると、`NotebookDirectory[]` 内に対応するフォルダ・ファイルが作成されます

```mathematica
(* API から直接呼び出す例 *)
DocExportMarkdown[EvaluationNotebook[]]
DocExportLaTeX[EvaluationNotebook[]]
DocExportLaTeX[EvaluationNotebook[], "MathFormat" -> True]  (* 数式自動フォーマット *)
DocExportWord[EvaluationNotebook[]]
DocExportWord[EvaluationNotebook[], "MathFormat" -> True]   (* 数式自動フォーマット *)
DocExportWord[EvaluationNotebook[], "ReferenceDoc" -> "/path/to/template.docx"]
```

## セル状態の遷移

各セルは以下の状態を持ちます。アイデアメモとパラグラフは常に両方保持されており、表示を切り替えるだけで元に戻れます。

```
idea（アイデアメモ）
  ↓ ▶ 展開
paragraph（パラグラフ）
  ↓ » 翻訳
translated（翻訳済み）

idea（プロンプト）
  ↓ 計算
compute（生成コード表示）

idea ↔ paragraph      （↔ 切替ボタン）
paragraph ↔ translated （↔ 切替ボタン）
translated → paragraph → idea （切替の逆方向も可）
computePrompt ↔ compute （↔ 切替ボタン）

paragraph → paragraph（更新）  （▶ 展開：展開済みセルへの再実行）
translated → translated（更新） （» 翻訳：翻訳済みセルへの再実行）

paragraph / idea → （素のテキストセル）  （×展開）
translated → paragraph              （×翻訳）
compute → computePrompt             （×計算）
```

アイデアテキストはセルの TaggingRules（Mathematica 内部メタデータ）に保存されるため、ノートブックを閉じて再度開いても状態は保持されます。

**メタセル（Note・Directive・Dictionary・Bibliography）および除外設定されたセルは上記の状態機械の外にあります。** これらのセルは展開・翻訳・切替の対象にならず、エクスポート時にも本文には出力されません。

## 公開 API リファレンス

通常はパレットのボタン操作で十分ですが、スクリプトや自動化のために公開 API を直接呼び出すこともできます。

### 執筆ツール

#### DocExpandIdea

選択したセルのアイデアテキストを LLM を使って文章品質のパラグラフに展開します。
既にパラグラフ展開済みの場合は、現在のパラグラフ文章・Directive セルの指示・文脈を踏まえてパラグラフをインプレースで更新します。

**シグネチャ**

```mathematica
DocExpandIdea[nb, cellIdx]
DocExpandIdea[nb, cellIdx, Fallback -> True]
```

| 引数 | 型 | 説明 |
|---|---|---|
| `nb` | `NotebookObject` | 対象ノートブック |
| `cellIdx` | `Integer` | 対象セルのインデックス（1 始まり） |
| `Fallback` | `True` / `False` | 課金 API（Claude API）を許可するか。デフォルト `False` |

**例**

```mathematica
(* 現在のノートブックの 3 番目のセルを展開 *)
DocExpandIdea[EvaluationNotebook[], 3]

(* 課金 API を許可して展開（既展開セルへの再実行も同様） *)
DocExpandIdea[EvaluationNotebook[], 3, Fallback -> True]
```

---

#### DocToggleView

セルのアイデア表示とパラグラフ表示を切り替えます。
現在表示中の内容（編集済みでも）を保存してから切り替えます。
翻訳済みセルの場合はパラグラフ ↔ 翻訳を、計算モードのセルの場合はプロンプト ↔ コードを切り替えます。

**シグネチャ**

```mathematica
DocToggleView[nb, cellIdx]
```

| 引数 | 型 | 説明 |
|---|---|---|
| `nb` | `NotebookObject` | 対象ノートブック |
| `cellIdx` | `Integer` | 対象セルのインデックス |

**例**

```mathematica
(* 5 番目のセルのアイデア ↔ パラグラフを切り替え *)
DocToggleView[EvaluationNotebook[], 5]
```

---

#### DocTranslate

セルの内容を `$DocTranslationLanguage` で指定した言語に翻訳します。
パラグラフモードのセルが対象です（アイデアモードのセルはスキップされます）。
既に翻訳済みのセルに対して呼び出した場合は、現在のパラグラフから翻訳を再生成します。

**シグネチャ**

```mathematica
DocTranslate[nb, cellIdx]
DocTranslate[nb, cellIdx, Fallback -> True]
```

**例**

```mathematica
(* 2 番目のセルを翻訳（翻訳済みの場合は再翻訳） *)
DocTranslate[EvaluationNotebook[], 2]
```

---

#### DocSync

翻訳済みセルの元テキストと翻訳を再同期します。
元テキストが編集された場合に、翻訳全体を再生成せず差分的に更新します。

> **注意**: パレットには同期ボタンがありません。スクリプトや自動化での利用を想定した API です。軽微な変更には本関数が効率的ですが、大幅な変更の場合は `DocTranslate` で全体を再翻訳することを推奨します。

**シグネチャ**

```mathematica
DocSync[nb, cellIdx]
DocSync[nb, cellIdx, Fallback -> True]
```

**例**

```mathematica
(* 4 番目のセルを同期 *)
```

#### DocCompute

選択したセルのプロンプトテキストから、LLM を使って実行可能な Wolfram Language / Mathematica コードを生成します。
ノートブックのコンテキスト（文脈情報）を自動収集し、略語や固有名詞の意味を文脈から判断します。
生成されたコードはセルに書き込まれ、セルはオレンジ色の枠線の計算モードに切り替わります。
ノートブック内の Directive セルの指示もコード生成時に適用されます。

**シグネチャ**

```mathematica
DocCompute[nb, cellIdx]
DocCompute[nb, cellIdx, Fallback -> True]
```

| 引数 | 型 | 説明 |
|---|---|---|
| `nb` | `NotebookObject` | 対象ノートブック |
| `cellIdx` | `Integer` | 対象セルのインデックス（1 始まり） |
| `Fallback` | `True` / `False` | 課金 API（Claude API）を許可するか。デフォルト `False` |

**例**

```mathematica
(* 現在のノートブックの 3 番目のセルのプロンプトからコードを生成 *)
DocCompute[EvaluationNotebook[], 3]

(* 課金 API を許可してコードを生成 *)
DocCompute[EvaluationNotebook[], 3, Fallback -> True]
```

> **注意**: コード表示中（計算モード）のセルに対して呼び出すことはできません。先に `DocToggleView` でプロンプト表示に切り替えてから実行してください。

#### DocExportMarkdown

### 設定変数

#### $DocTranslationLanguage

翻訳先の言語名を指定するグローバル変数です。

- デフォルト: `$Language` が英語以外なら `"English"`、英語なら `"Japanese"`
- ユーザーが任意の言語名に変更可能です

**例**

```mathematica
(* フランス語に翻訳する *)
$DocTranslationLanguage = "French"

(* 日本語に翻訳する *)
$DocTranslationLanguage = "Japanese"
```

---

#### $ClaudeModel（ClaudeCode パッケージ変数）

使用する LLM モデルを指定します。このパッケージの LLM 呼び出しはすべて claudecode パッケージ経由で行われるため、`ClaudeCode`$ClaudeModel` の設定が適用されます。

- **文字列を設定した場合**: Anthropic API を使用します（例: `"claude-sonnet-4-20250514"`）
- **リストを設定した場合**: ローカル LMStudio モデルを使用します（後述の `$ClaudePrivateModel` 参照）

---

#### $ClaudePrivateModel（ClaudeCode パッケージ変数）

LMStudio ローカルモデルの接続設定を `{プロバイダ種別, モデル名, ベースURL}` の 3 要素リストで指定します。

```mathematica
$ClaudePrivateModel = {"lmstudio", "qwen/qwen3.6-27b", "http://127.0.0.1:1234"}
```

| 要素 | 説明 |
|---|---|
| 第1要素 | プロバイダ種別。`"lmstudio"` を指定します |
| 第2要素 | LMStudio で起動しているモデル名 |
| 第3要素 | LMStudio ローカルサーバーのベース URL（デフォルト: `"http://127.0.0.1:1234"`） |

`$ClaudeModel = $ClaudePrivateModel` と設定することでローカルモデルに切り替わります。

---

#### $ClaudeLMStudioIntegrations（ClaudeCode パッケージ変数）

LMStudio モード時に有効化する MCP ツール統合のリストを指定します。

```mathematica
$ClaudeLMStudioIntegrations = {"mcp/exa"}
```

- `"mcp/exa"` を指定すると Exa 検索 MCP が LMStudio モードで利用可能になります
- 空リスト `{}` を指定すると MCP 統合なしで動作します

## 注意事項

- パレットは `WindowFloating` モードで表示されます。パレットをクリックしても対象ノートブックの選択は変わりません。
- セルを選択せずにボタンを押した場合、直前に操作したセルが再利用されます。
- 複数セルの逐次処理は約 2 秒間隔の非同期チェーンで実行されます。処理中はステータスバーに進捗が表示されます。
- モデル設定は `ClaudeCode` パレットと共有されます。片方で変更すると両方のパレットに反映されます。
- エクスポート機能（Markdown・LaTeX・Word）は試験的な実装です。数式・複雑な表・埋め込みオブジェクトなど、一部の要素が正確に変換されない場合があります。
- **メタセル（Note・Directive・Dictionary・Bibliography）および除外設定されたセルは、展開・翻訳・同期・一括表示切替の対象外であり、Markdown・LaTeX・Word エクスポート時にも本文に含まれません。**
- Directive セルと Dictionary セルはノートブック内に複数配置できます。すべての Directive・Dictionary セルの内容が LLM に渡されます。Directive セルは計算（コード生成）時にも参照されます。
- `<<fig:label>>` および `<<cite:key>>` マーカーはエクスポート時に自動変換されます。本文中に直接記述してください。
- **×展開・×翻訳・×計算は元に戻せません。** 実行前に確認ダイアログが表示されますが、削除後のデータは復元できないため注意してください。
- Word エクスポートには Pandoc が必要です。インストール方法は `setup.md` を参照してください。
- `DocAutoInsertCitations` は依存資料（Ref Src）が設定されたセルがある場合のみ動作します。先に各セルへ依存資料を設定してください。
- +Math（数式自動フォーマット）オプションは、テキスト中にインラインで書かれた数式表現（例: 「f(x) = x^2」）のほか、Box 式・FormBox を含む Mathematica の数式表現も対象としています。Mathematica の StandardForm 数式セルとは別の処理です。
- **▶ 展開 を展開済みセルに再実行すると、アイデアメモからではなく現在のパラグラフ文章・指示・文脈を踏まえてインプレース更新されます。** Directive や Dictionary を変更した後に既存パラグラフへ反映させたい場合に活用してください。
- **» 翻訳 を翻訳済みセルに再実行すると、現在のパラグラフから翻訳を再生成します。** 元パラグラフの変更が軽微な場合は `DocSync[nb, cellIdx]` API で差分更新できます。
- **計算（コード生成）はコード表示中のセルに再実行できません。** **↔ 切替** でプロンプト表示に戻してからプロンプトを修正し、再度 **計算** を実行してください。生成されたコードはオレンジ色の枠線で表示されます。
- LaTeX エクスポート時の画像は本文幅いっぱいに出力されます（`\includegraphics[width=\textwidth]`）。
- **LMStudio ローカルモデルを使用する場合は、パッケージロード前に LMStudio のローカルサーバーを起動しておいてください。** `$ClaudeModel` をリストに設定している間は、パレットの課金 API 設定や Fallback オプションに関わらず LMStudio が優先されます。
